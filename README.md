# MSPM0G3519 Competition Firmware Template

Firmware template for TI MSPM0G3519 electronic-design competition projects.
The target is the `LQFP-80(PN)` package and the project uses TI DriverLib.
It includes SysConfig, Keil MDK, IAR, TI Clang, and CMake/VS Code entry points.

## Target And Tools

- MCU: `MSPM0G3519`
- Package: `LQFP-80(PN)`
- SDK: `MSPM0 SDK 2.10.00.04`
- SysConfig: `1.28.0`
- TI Arm Clang: `4.0.2.LTS`

Expected local tool paths:

```text
C:\ti\mspm0_sdk_2_10_00_04
C:\ti\sysconfig_1.28.0
C:\ti\ti_cgt_arm_llvm_4.0.2.LTS
```

## Project Layout

```text
Algorithm/     Attitude estimation, Kalman, Mahony, and quaternion EKF
apply/         Encoder, gray sensor, IMU, motor control, and timing code
Drive/         Board drivers: LCD, SPI, UART, motor, key, buzzer, and delay
Task/          Application tasks
keil/          Keil MDK project
iar/           IAR project
ticlang/       TI Clang projectspec project
gcc/           GCC projectspec project
empty.syscfg   SysConfig source with the current 80-pin pin assignments
```

## SysConfig

Use the `SysConfig` status-bar button in VS Code, or run:

```bat
open_sysconfig_128.bat
```

Always open and save `empty.syscfg`. It contains the MSPM0G3519 target,
the LQFP-80 package, and the configured peripheral pins. Do not save a
default 100-pin SysConfig page over this file.

## CMake And VS Code

Install CMake Tools and clangd, then run from the repository root:

```powershell
cmake --preset mspm0-debug
cmake --build build/mspm0-debug --target firmware_objects
```

The generated `compile_commands.json` supplies the compiler flags, include
paths, and MSPM0G3519 definitions used by clangd.

## Keil

Open `keil/empty_LP_MSPM0G3519_nortos_keil.uvprojx`. Before the first build,
verify that the configured TI Arm Clang and MSPM0 SDK paths match the local
installation.

## Notes

- `ti_msp_dl_config.c` and `ti_msp_dl_config.h` are generated by SysConfig.
- Save SysConfig after changing pins, clocks, or peripherals.
- Build directories, Keil output files, and SysConfig local state are ignored
  by Git.
